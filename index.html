Mapzen Styleguide makes it easy for us to create right-looking websites that work well across platforms and into the future. If you’re publishing Mapzen stuff, the guide is for you.

Batteries included:

  • Basic page structure with standard linked CSS and Javascript resources.
  • Sample UI components like navigation, social buttons, search boxes, and pagination that can be combined to make new kinds of pages.
  • Complete sample pages like blog posts and developer documentation showing how all the pieces fit together.

Get started

Find the Mapzen Styleguide on GitHub at mapzen/styleguide.

HTML boilerplate

To make a Mapzen-branded page, copy the HTML below. This creates the basic page structure ready to receive your content.

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <meta http-equiv="x-ua-compatible" content="ie=edge">
  <title>[YOUR PAGE TITLE] · Mapzen</title>
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <link rel="stylesheet" href="https://mapzen.com/common/styleguide/styles/styleguide.css">
</head>
<body>
  <div class="container">
    <div class="row">
      <div class="col-md-8 col-md-offset-2">
        <h1>Hello world!</h1>
        <p>This is Mapzen Styleguide: HTML Boilerplate.</p>
      </div>
    </div>
  </div>
  <script src="https://mapzen.com/common/styleguide/scripts/mapzen-styleguide.min.js"></script>
</body>
</html>

Browser support

We aim to support the following browsers and devices:

  • Browsers
    • Chrome (latest) — Mac OSX / Windows
    • Firefox (latest) — Mac OSX / Windows
    • Safari (latest) — Mac OSX only
    • Internet Explorer (version 9+) — Windows only
  • Devices (tbd)
    • iPhone 5+
    • iPad 2
    • Android

We’ll review browser and device usage about every six months and adjust this list as necessary. We plan to support browsers and devices that are still used by a certain threshold of users and viewers.

Tangram requires WebGL, which is not supported in Internet Explorer 9 and 10. WebGL can also be disabled by an administrator. If a map or demo uses Tangram but WebGL is unavailable, please provide a fallback raster map (if possible), a link to an alternative map, or a helpful message explaining why the map cannot be viewed.

Testing

We have a shared Browserstack account for testing on all supported browsers, platforms and devices.

You can also test Internet Explorer locally. We recommend using Microsoft’s virtual machines in your VM platform of choice (e.g. VirtualBox).
