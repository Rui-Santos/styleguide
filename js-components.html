JS Components

Some components will require JavaScript in order to function correctly. In addition, we assume that all of our pages will be measured by Google Analytics. So, we've bundled all of the baseline JavaScript requirements into one package: 

<script src="https://mapzen.com/common/styleguide/scripts/mapzen-styleguide.min.js"></script>

This file is about 50kb minified and gzipped. For best results, insert this script tag at the bottom of a webpage, just before the closing </body> tag. You can see its preferred placement in the HTML Boilerplate.

These third-party libraries are included:

  • Bootstrap JavaScript (version 3.3.6) for these plugins: Transitions, Dropdown, Scrollspy, Collapse, and Affix. Other Bootstrap plugins are not supported by our styleguide at this time.
  • Google Analytics using Mapzen's Universal Analytics ID. We also include a helper function for tracking specific events (documentation forthcoming).

In addition, client-side JavaScript is used to make sure our top navigation bar works correctly, and to handle very wide tables. Specific design components that depend on JavaScript are described in greater detail below. If you need other libraries that are not included in this list above, please include them in your page separately. (For instance, jQuery is not provided by the Styleguide.)

Polyfills

We include some polyfills for the latest standard JavaScript APIs, so that you can write modern code that will continue to work in older browsers that we support. Not all modern JavaScript is polyfilled, but we include the following:

  • Object.assign() via object-assign.
  • Promise via promise.
  • fetch() via whatwg-fetch.
  • HTMLElement.classList via classlist-polyfill.
  • Array.isArray() via MDN.
  • <input type="range"> via rangeslider.js.

Please note that polyfills are immediately available in global scope and added as a side effect of including this bundle. If the browser already supports a feature natively, polyfills will defer to the native functionality.

Side Navigation

The sub-navigation is used when there are multiple pages in a section. It can have up to three levels of information.

This component will take up 100% width of its parent container. To put it on the side of a page, place it inside a grid containing element, such as .col-sm-2 for a two-column wide navigation element. On mobile screens, this component should be the full width of the device.

  • Use the ARIA roles 'directory', 'navigation', or 'complementary' where appropriate
  • Use the .affix-* classes to force it to never scroll off the page
  • The .nav base class is from Bootstrap and some functionality is inherited from it
